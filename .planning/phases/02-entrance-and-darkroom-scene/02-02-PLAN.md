**initEntrance() must do these things in order:**

1. **"Step Inside" CTA fade-in (ENTR-03):**
   ```javascript
   gsap.to('.entrance__cta', {
     opacity: 1,
     duration: 1.5,
     delay: 1.5,
     ease: 'power2.out'
   });
   ```
   The delay is 1.5s per user decision ("fade-in after 1-2 second delay"). The text uses Playfair Display via CSS.

2. **Door hover glow (ENTR-04):**
   Wire `mouseenter` and `mouseleave` on `#door`:
   - mouseenter: `gsap.to(door, { filter: 'drop-shadow(0 0 20px rgba(255,140,0,0.7))', duration: 0.3 })` and set `cursor: pointer`
   - mouseleave: `gsap.to(door, { filter: 'none', duration: 0.4 })`

   Also wire keyboard Enter key on the door (it has `role="button" tabindex="0"`):
   ```javascript
   door.addEventListener('keydown', (e) => {
     if (e.key === 'Enter') door.click();
   });
   ```

3. **Build walk-in GSAP timeline (ENTR-05, ENTR-06) — PAUSED:**
   Create the entire timeline paused. The timing should total 3-4 seconds per user decision ("slow and cinematic pacing").

   ```javascript
   const walkIn = gsap.timeline({
     paused: true,
     defaults: { ease: 'power2.inOut' },
     onComplete: () => {
       store.set({ transitionInProgress: false });
       showDarkroom();
     }
   });
   ```

   **Phase A: Approach (0s-1.5s)** — Layers surge toward viewer, creating forward-motion feeling:
   - Ground layer shifts down (`y: '25%'`, `scale: 1.2`, duration 1.5s)
   - Building layer scales up (`scale: 1.15`, duration 1.5s)
   - Door-frame layer scales up (`scale: 1.3`, duration 1.5s)
   - Door layer scales up dramatically (`scale: 1.6`, duration 1.5s)
   - CTA text fades out immediately (`opacity: 0`, duration 0.4s)
   - Light leak fades out (`opacity: 0`, duration 0.5s)
   All approach tweens start at position 0 (parallel).

   **Phase B: Darkness (1.2s-1.9s)** — Overlapping with end of approach:
   - Darkness overlay fades to full opacity (`opacity: 1`, duration 0.7s, starts at position 1.2)

   **Phase C: Hold (1.9s-2.3s)** — Brief darkness hold:
   - Keep darkness at opacity 1 for 0.4s (use `.to(darkness, { opacity: 1, duration: 0.4 }, 1.9)`)

   Total timeline: ~2.3s, then onComplete triggers darkroom reveal which takes another ~1.5s = ~3.8s total. Within the 3-4 second user decision window.

4. **Door click handler:**
   ```javascript
   door.addEventListener('click', () => {
     if (store.get().transitionInProgress) return; // double-click guard
     store.set({ transitionInProgress: true });
     walkIn.play();
   });
   ```

5. **showDarkroom() function (internal, not exported):**
   This transitions from the entrance to the darkroom:
   ```javascript
   function showDarkroom() {
     const entrance = document.getElementById('entrance');
     const darkroom = document.getElementById('darkroom');

     gsap.set(entrance, { display: 'none' });
     darkroom.style.display = 'block';
     gsap.set(darkroom, { opacity: 0 });

     gsap.to(darkroom, {
       opacity: 1,
       duration: 1.5,
       ease: 'power2.out',
       onComplete: () => {
         gsap.set('.scene__overlay--darkness', { opacity: 0 });
         revealDarkroom(); // stagger photos in + enable interactions
       }
     });
   }
   ```

**CRITICAL anti-patterns to avoid (from research):**
- Do NOT create the timeline inside the click handler — build once, play on click
- Do NOT animate `left`/`top` — use GSAP `x`/`y`/`scale` (transform-based)
- Do NOT call `navigateTo()` — this is a visual transition, not a route change
- Do NOT use `100vh` without `100dvh` fallback (handled in CSS, not JS concern)
- The darkness overlay MUST have `pointer-events: none` in CSS (already set in Plan 01)

1. **Nav element hover glows (DARK-08, DARK-10):**
   Use a helper function for DRY code:
   ```javascript
   function wireNavElement(elementId, labelId, route) {
     const el = document.getElementById(elementId);
     const label = document.getElementById(labelId);
     if (!el || !label) return;

     el.addEventListener('mouseenter', () => {
       gsap.to(el, { filter: 'drop-shadow(0 0 15px rgba(255,140,0,0.6))', duration: 0.3 });
       gsap.to(label, { opacity: 1, duration: 0.3 });
     });
     el.addEventListener('mouseleave', () => {
       gsap.to(el, { filter: 'none', duration: 0.4 });
       gsap.to(label, { opacity: 0.5, duration: 0.4 });
     });
     el.addEventListener('click', () => navigateTo(route));
     // Keyboard accessibility
     el.addEventListener('keydown', (e) => {
       if (e.key === 'Enter') navigateTo(route);
     });
   }
   ```

   Wire both nav elements:
   ```javascript
   wireNavElement('silhouette-figure', 'label-about', '/about');
   wireNavElement('business-card', 'label-contact', '/contact');
   ```

2. **Clothesline photo hover — subtle scale lift (preview for Phase 3 hand-grab):**
   Add mouseenter/mouseleave to each `.clothesline__photo`:
   ```javascript
   document.querySelectorAll('.clothesline__photo').forEach(photo => {
     photo.addEventListener('mouseenter', () => {
       gsap.to(photo.querySelector('.clothesline__image'), {
         scale: 1.05,
         boxShadow: '0 8px 24px rgba(0,0,0,0.8)',
         duration: 0.3
       });
     });
     photo.addEventListener('mouseleave', () => {
       gsap.to(photo.querySelector('.clothesline__image'), {
         scale: 1,
         boxShadow: '0 4px 12px rgba(0,0,0,0.6)',
         duration: 0.4
       });
     });
     // Photo click is a no-op for now — Phase 3 adds the hand-grab transition
     photo.addEventListener('click', () => {
       const genre = photo.dataset.genre;
       console.log(`[darkroom] Photo clicked: ${genre} (hand-grab transition in Phase 3)`);
     });
   });
   ```

**revealDarkroom() — called after walk-in transition completes:**
This provides the staggered photo entrance animation (DARK-03) that plays when the darkroom first becomes visible:

```javascript
export function revealDarkroom() {
  // Stagger-reveal the clothesline photos
  gsap.from('.clothesline__photo', {
    opacity: 0,
    y: -20,
    duration: 0.8,
    stagger: {
      each: 0.15,
      from: 'start',
      ease: 'power2.out'
    },
    ease: 'power2.out'
  });

  // Fade in nav elements slightly after photos
  gsap.from('.nav-element', {
    opacity: 0,
    y: 10,
    duration: 0.6,
    delay: 1.0,
    stagger: 0.2,
    ease: 'power2.out'
  });
}
```

**IMPORTANT:** `initDarkroom()` is for setup (event listeners). `revealDarkroom()` is for the one-time entrance animation. They are separate because `initDarkroom` runs on page load (wiring), while `revealDarkroom` runs only after the walk-in transition ends.

**Mobile considerations (ENTR-07):**
The hover effects use mouseenter/mouseleave which do not fire on mobile touch — this is correct behavior. Touch users tap to navigate or select photos. The CSS handles responsive sizing. No additional mobile JS needed here.
