  <name>Task 1: Create state store and router modules</name>
  <files>js/store.js, js/router.js</files>
  <action>
**js/store.js** — Minimal pub/sub state store per research Pattern 3:
- Internal `state` object with properties:
  - `currentRoute: '/'`
  - `currentGenre: null`
  - `selectedPhotoIndex: null`
  - `transitionInProgress: false`
  - `genres: []`
- `listeners` as a `new Set()`
- Export `store` object with three methods:
  - `get()` — returns shallow copy of state (`{ ...state }`)
  - `set(updates)` — merges updates via `Object.assign(state, updates)`, then calls each listener with a shallow copy
  - `subscribe(fn)` — adds fn to listeners, returns an unsubscribe function that deletes fn from the set

**js/router.js** — History API SPA router per research Pattern 2:
- Import `store` from `'./store.js'`
- Export `initRouter()`:
  - Add a `click` event listener on `document` that:
    - Finds the closest `a[href]` ancestor of the click target
    - Skips if no link found, or if link has `target="_blank"`, or if href starts with `http` (external links)
    - For internal links (starts with `/` or is relative): `e.preventDefault()`, call `navigateTo(href)`
  - Add a `popstate` event listener on `window` that:
    - Gets `window.location.pathname`
    - Updates store: `store.set({ currentRoute: path })`
    - Dispatches `new CustomEvent('route:change', { detail: { path, state: e.state, trigger: 'popstate' } })` on window
    - Logs to console: `[router] popstate → ${path}`

- Export `navigateTo(path, state = {})`:
  - Call `history.pushState({ path, ...state }, '', path)`
  - Update store: `store.set({ currentRoute: path })`
  - Dispatch `new CustomEvent('route:change', { detail: { path, state, trigger: 'push' } })` on window
  - Log to console: `[router] navigateTo → ${path}`

CRITICAL per research: pushState does NOT fire popstate. The manual CustomEvent dispatch after pushState is essential — this is not redundant, it's required by the HTML spec.

Do NOT call navigateTo or pushState during page load (per research anti-pattern — breaks WebKit back button).

**js/main.js** — App entry point (ES module):

1. Import `{ initRouter }` from `'./router.js'`
2. Import `{ store }` from `'./store.js'`

3. Register GSAP plugins (globals from CDN):
   ```javascript
   gsap.registerPlugin(ScrollTrigger, ScrollSmoother);
   ```
   Log: `[main] GSAP ${gsap.version} registered with ScrollTrigger, ScrollSmoother`

4. Create `async function loadGallery()`:
   - `fetch('./gallery.json')` with error handling (try/catch)
   - Validate response: if `!res.ok`, throw with HTTP status
   - Parse JSON, validate `data.genres` is an array
   - Store in state: `store.set({ genres: data.genres })`
   - Log: `[gallery] Loaded ${data.genres.length} genres`
   - Return data

5. Create `function init()`:
   - Call `initRouter()`
   - Call `await loadGallery()`
   - Subscribe to store for debugging: `store.subscribe(state => console.log('[store] state updated:', state))`
   - Log: `[main] Darkroom Portfolio initialized`
   - Set the initial route in store: `store.set({ currentRoute: window.location.pathname })`

6. Add a `route:change` event listener on window that logs: `[main] Route changed: ${e.detail.path} (${e.detail.trigger})`

7. Wrap `init()` in DOMContentLoaded:
   ```javascript
   document.addEventListener('DOMContentLoaded', init);
   ```

8. Add temporary test navigation links to the #app div for verification (will be removed in Phase 2):
