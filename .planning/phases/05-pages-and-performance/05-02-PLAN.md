**index.html — add preload links in `<head>`:**

After the CSS stylesheet links and before `</head>`, add preload hints for the 5 top-clothesline background images. These are above-fold images rendered via CSS `background-image` — the browser doesn't discover them until CSS is parsed, so preloading gives them a head start.

```html
<!-- Preload clothesline thumbnails (above-fold background images) -->
<link rel="preload" as="image" href="/images/fashion/high-fashion-gobo-Photography.jpg" fetchpriority="high">
<link rel="preload" as="image" href="/images/beauty/DSC5447-Edit-min.jpg">
<link rel="preload" as="image" href="/images/light-painting/Russell-Klimas-Lightpainting-min.jpg">
<link rel="preload" as="image" href="/images/drone/Balance-Steamboat-Garden-of-the-Gods-drone-light-painting.jpg">
<link rel="preload" as="image" href="/images/gels/DSC7601-Edit.jpg">
```

Use `fetchpriority="high"` only on the first image (fashion) since it's the leftmost and likely LCP candidate. The other 4 use default priority.

IMPORTANT (Pitfall 2): The `href` paths MUST use root-relative `/images/...` format. The inline `style` attributes in index.html currently use relative paths (`url('images/...')`). Both resolve to the same file on the server, but the preload href must be consistent. Verify each filename exactly matches the corresponding clothesline photo's background-image URL.

**js/gallery.js — verify lazy loading:**

Read gallery.js and confirm that:
1. `createFrame()` includes `loading="lazy"` on `<img>` elements — this is already implemented.
2. `buildFilmstrip(genre)` is only called from `transition.js` onComplete callback (after forward transition finishes) — meaning filmstrip images are NOT fetched until a genre is actually visited.
3. If both conditions are confirmed, PERF-03 is already satisfied. No code changes needed in gallery.js.
