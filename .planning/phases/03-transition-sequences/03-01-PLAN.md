<task type="auto">
  <name>Task 2: Create transition.js — forward and reverse animation controller</name>
  <files>js/transition.js</files>
  <action>
Create `js/transition.js` as an ES module exporting `startForward(genre, photoEl)` and `startReverse()`.

**Imports:**
```javascript
import { store } from './store.js';
import { navigateTo } from './router.js';
```

GSAP is a CDN global — do NOT import it.

**Module-level state:**
- `let forwardTl = null;` — the stored forward timeline (rebuilt per click since photo position varies)
- `let activePhotoEl = null;` — reference to the currently transitioned photo element

**`startForward(genre, photoEl)` function:**

1. Guard: if `store.get().transitionInProgress` is true, return immediately.
2. Set store: `{ transitionInProgress: true, currentGenre: genre }`.
3. Pause CSS sway on ALL `.clothesline__photo` elements: `el.style.animationPlayState = 'paused'`.
4. Snap the grabbed photo to neutral rotation: `gsap.set(photoEl, { rotate: 0 })`.
5. Measure the photo's position: `const rect = photoEl.getBoundingClientRect()`. Calculate scale, dx, dy per RESEARCH.md Pattern 2 (scale-to-fill).
6. Get DOM references: `handEl = document.getElementById('hand-silhouette')`, `pinEl = photoEl.querySelector('.clothesline__pin')`, `darkroomEl = document.getElementById('darkroom')`, `galleryEl = document.getElementById('gallery')`.
7. Store `activePhotoEl = photoEl`.
8. Prep the gallery placeholder: set the `.gallery__genre-name` text to the genre name (capitalize first letter). Add class `gallery--visible` to `#gallery`. Set `gsap.set(galleryEl, { display: 'flex', opacity: 0 })`.
9. Build `forwardTl = gsap.timeline({ ... })`:

**ACT 1 — GRAB (0 to ~1.2s):**
- Hand enters from bottom: `gsap.set(handEl, { display: 'block', y: '100vh', opacity: 1 })`, then `.to(handEl, { y: targetY, duration: 0.8, ease: 'power2.out' }, 0)` where `targetY` positions the hand near the clothespin (calculate from `rect.top` — the hand should reach up to where the pin is).
- Clothespin releases at ~0.6s: `.to(pinEl, { y: -8, opacity: 0, duration: 0.3, ease: 'power2.in' }, 0.6)`.
- Neighbor photos sway (disturbance) at ~0.7s: get siblings of `photoEl` within the same `.clothesline` parent, filter out decorative. `.to(neighbors, { rotate: (i) => (i % 2 === 0 ? 3 : -3), duration: 0.4, ease: 'power2.out', yoyo: true, repeat: 1 }, 0.7)`.
- Clothesline wire bounce at ~0.7s: `.to(photoEl.closest('.clothesline').querySelector('.clothesline__wire'), { y: 2, duration: 0.2, ease: 'power2.out', yoyo: true, repeat: 1 }, 0.7)`.
- Hand retreats at ~0.9s: `.to(handEl, { y: '100vh', opacity: 0, duration: 0.4, ease: 'power2.in' }, 0.9)`.
- After hand retreats, hide it: `.set(handEl, { display: 'none' }, 1.3)`.

**ACT 2 — PULL-IN (~0.9 to ~2.1s):**
- Photo scales to fill viewport: `.to(photoEl, { scale, x: '+=' + dx, y: '+=' + dy, transformOrigin: '50% 50%', duration: 1.2, ease: 'power2.inOut' }, 0.9)`.
- Darkroom fades behind: `.to(darkroomEl, { opacity: 0, duration: 0.8, ease: 'power2.in' }, 1.4)`.

**ACT 3 — DISSOLVE (~1.8 to ~2.1s):**
- Gallery fades in: `.to(galleryEl, { opacity: 1, duration: 0.3, ease: 'none' }, 1.8)`.
- Hide darkroom after fade: `.set(darkroomEl, { display: 'none' }, 2.1)`.

**onComplete callback:**
- `store.set({ transitionInProgress: false })`.
- `photoEl.classList.add('clothesline__photo--empty')`.
- Call `navigateTo('/gallery/' + genre)` to update the URL bar.

10. Play the timeline (it is NOT paused — builds and plays immediately).

**`startReverse()` function:**

1. Guard: if `!forwardTl` or `store.get().transitionInProgress`, return.
2. Set `store.set({ transitionInProgress: true })`.
3. Ensure hand is hidden before reversing (per user decision — no hand replay on return): `gsap.set(document.getElementById('hand-silhouette'), { display: 'none', y: '100vh' })`.
4. Ensure darkroom is display block and opacity 0 so it can fade back in: `gsap.set(document.getElementById('darkroom'), { display: 'block' })`.
5. Set `forwardTl.timeScale(2)` then call `forwardTl.reverse()`.
6. Set `forwardTl.eventCallback('onReverseComplete', () => { ... })`:
   - Remove `clothesline__photo--empty` from `activePhotoEl`.
   - `gsap.set(activePhotoEl, { clearProps: 'scale,x,y,rotate,opacity,transform' })` to reset GSAP transforms.
   - Reset pin: `gsap.set(activePhotoEl.querySelector('.clothesline__pin'), { clearProps: 'y,opacity' })`.
   - Resume CSS sway on all non-empty photos: `el.style.animationPlayState = 'running'`.
   - Remove `gallery--visible` class from gallery, set gallery `display: 'none'`.
   - Reset `forwardTl.timeScale(1)`.
   - `store.set({ transitionInProgress: false, currentGenre: null })`.
   - `activePhotoEl = null`.
   - `forwardTl = null` (allow fresh timeline on next click since photo may be different).

**Helper function `getNeighborPhotos(photoEl)`:**
Get all `.clothesline__photo` siblings within the same `.clothesline` parent, excluding `photoEl` itself and decorative photos.

**Important implementation notes from RESEARCH.md:**
- ALWAYS pause CSS sway via `animationPlayState = 'paused'` BEFORE any GSAP transform on clothesline photos (Pitfall 1).
- ALWAYS snap to `rotate: 0` and THEN measure `getBoundingClientRect()` (Pitfall 2).
- The `transitionInProgress` guard prevents double-trigger (Pitfall 3).
- Hand SVG z-index 100 prevents stacking context issues (Pitfall 4).
- Hand is hidden via `display: none` before `reverse()` to prevent hand replay (Pitfall 5).
- Gallery is prepped (display: flex, opacity: 0) before the timeline starts to avoid first-render stutter (Pitfall 6).
- `onReverseComplete` restores the absent photo and clears GSAP props (Pitfall 7).
